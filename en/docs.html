The Complete Reference v2.0

Welcome to the ultimate guide for Audio XML (AXML). This specification defines a robust, text-based format for music synthesis and composition on the web.

Whether you are a sound designer code-sculpting a lead synth or a composer building a complex arrangement, AXML gives you precise control over every frequency and beat.

account_tree Root Hierarchy

A valid AXML file must follow this exact order of elements. The engine parses them sequentially to build the audio graph.

<?xml version="1.0" encoding="UTF-8"?>
<axml version="1.0">
  <metadata> ... </metadata>
  <instruments> ... </instruments>
  <patterns> ... </patterns>
  <tracks> ... </tracks>
</axml>

info Global Metadata

Metadata defines the project context. tempo is critical as it defines how beats translate to seconds.

Element Description
<title> Display name of the project.
<artist> Project creator or sound designer.
<tempo> Beats Per Minute (60.0 - 250.0). Average: 120.
<key> Musical key signature (e.g., C, Am, F#m).
<time-signature> Rhythm structure (e.g., 4/4, 3/4). 
<metadata>
  <title>Cyberpunk City</title>
  <artist>AXML Master</artist>
  <tempo>128</tempo>
  <key>Am</key>
</metadata>

waves Oscillator Types

The type attribute on <instrument> determines the basic waveform. AXML currently supports 11 distinct types:

  • synth
  • piano
    • Type Name Mathematical Waveform / Physical Model
    sine Pure sine wave (Fundamental frequency only).
    square Square wave (Rich odd harmonics).
    sawtooth Sawtooth wave (All harmonics, very bright).
    triangle Triangle wave (Mellow harmonics).
    kick Physical model: Sine sweep + Fast exponential decay.
    snare Physical model: White noise + Lowpass filter at 1.2kHz.
    hihat Physical model: White noise + Highpass filter at 8kHz.
    Alias for Sawtooth wave.
    Alias for Triangle wave.
    sample Plays an external audio file (requires src).

    library_music Audio Samples

    You can use external WAV, MP3, or OGG files as instruments. The engine will automatically decode and pitch-shift them based on the note pitch.

    <instrument id="vox" type="sample" 
    src="https://example.com/voice.wav" 
    volume="0.8" />
    Note: Samples are pitched relative to C4 (261.63Hz). If you play C4, the sample plays at original speed. C5 will play it at double speed.

    gesture ADSR Envelopes

    The volume envelope is critical for defining the "shape" of a sound. All values are in seconds, except sustain (0.0 to 1.0).

    Attribute Full Name Behavior
    attack Attack Time Seconds to reach peak volume.
    decay Decay Time Seconds to drop to sustain level.
    sustain Sustain Level Volume factor (0-1) held while note is active.
    release Release Time Seconds to fade to silence after note ends.

    filter_list Logic Filters

    Every instrument passes through a resonant Biquad Filter. This is essential for subtractive synthesis.

    Attribute Description Range
    cutoff The frequency above which sound is attenuated. 20Hz - 20,000Hz
    resonance Boosts the frequencies around the cutoff point. 0.1 - 20.0
    Subtractive Synthesis: Start with a rich waveform like sawtooth and use a low cutoff (e.g. 500) with high resonance to create a classic synth growl.

    auto_fix_high The FX Rack

    AXML Studio v2.0 features a complex chainable effects rack for every instrument.

    Effect Attribute Range Description
    Pan pan -1.0 to 1.0 -1.0 (Full Left) | 1.0 (Full Right)
    Distortion distortion 0.0 to 1.0 Soft-clipping saturation.
    Chorus chorus 0.0 to 1.0 Stereo widening and detune layering.
    Bitcrush bitcrush 0.0 to 1.0 Reduces bit-depth (16-bit to 2-bit lo-fi).
    Reverb reverb 0.0 to 1.0 Convolution-based spatial reverb.
    Delay delay 0.0 to 1.0 Wet amount for the echo delay.

    Delay also supports delayTime (seconds) and delayFeedback (repetition decay).

    vibration LFO System

    The Low Frequency Oscillator (LFO) automates parameter changes over time, creating movement and "wobble".

    Attribute Description
    lfoRate Speed of the oscillation in Hz.
    lfoAmount Intensity of the modulation.
    lfoType Shape of LFO (sine, square, sawtooth, triangle).
    lfoTarget Parameter to modulate (cutoff, volume, pan). 
    <!-- A classic Dubstep wobble -->
<instrument id="sub" type="sawtooth" 
    lfoRate="6" lfoAmount="0.8" lfoTarget="cutoff" />

    music_note Musical Notation

    AXML supports three types of pitch notation in the pitch attribute:

    • Scientific Notation: Use Letter + Accidental + Octave (e.g., C4, Eb5, G#3).
    • MIDI Note Numbers: Use integers (e.g., 60 for C4, 69 for A4).
    • Rests: Use the keyword rest for silence.

    timer Timing & Durations

    The time system in AXML is based on Beats, not seconds. 1 Beat = 1 Quarter Note.

    Duration Shortcuts

    w = 4 beats h = 2 beats q = 1 beat e = 0.5 beat s = 0.25 beat

    You can also use decimal values like duration="1.33" for custom timings.

    layers Chords & Polyphony

    There are two ways to play multiple notes simultaneously:

    1. The Chord Wrapper

    <chord start="0" duration="w">
  <note pitch="C4" />
  <note pitch="E4" />
  <note pitch="G4" />
</chord>

    2. Manual Start Timing

    <note pitch="C4" start="0" duration="w" />
<note pitch="E4" start="0" duration="w" />

    reorder Patterns & Macros

    AXML is designed for modularity. Patterns allow you to define complex musical phrases, drum loops, or chord progressions once and trigger them anywhere in your project using the <play /> element.

    Defining a Pattern

    All patterns must be defined inside the top-level <patterns> container. Each pattern requires a unique id.

    <patterns>
  <pattern id="drum_beat">
    <note pitch="C2" start="0" duration="0.5" velocity="1.0" /> <!-- Kick -->
    <note pitch="D4" start="1" duration="0.5" velocity="0.8" /> <!-- Snare -->
    <note pitch="C2" start="2" duration="0.5" velocity="1.0" />
    <note pitch="D4" start="3" duration="0.5" velocity="0.8" />
  </pattern>
</patterns>

    Invoking Patterns (Macros)

    Trigger a pattern inside a <track>. The start attribute on the <play> element acts as a time offset for all notes inside that pattern. 

    <track instrument="drums" name="Drums Section">
  <play pattern="drum_beat" start="0" />
  <play pattern="drum_beat" start="4" />
  <play pattern="drum_beat" start="8" />
</track>
    Nesting Patterns: AXML supports nested patterns. You can call a <play /> inside a <pattern> to build even more complex hierarchical structures (e.g. a Verse pattern that calls multiple Bar patterns).

    tune Master Dynamic Chain

    The AXML Engine doesn't just play tracks; it passes everything through a high-end master chain:

    1. Track Gains: Individual volume control for each track.
    2. Master Gain: Global volume control.
    3. Dynamics Compressor/Limiter: Prevents clipping and adds "glue" to the mix. Threshold set at -1.0 dB.
    4. Real-time Analyser: 512-band FFT for the visualizer engines.
